# Java 工具库模块 — 分析指导

## 分析模板

### 第一轮：结构扫描
- `pom.xml` → 外部依赖（第三方库如 Apache POI、Jackson 等）
- 目录结构 → 通常扁平，按功能分类
- `src/test/java` → 从测试用例了解使用方式

### 第二轮：核心代码扫描
- 所有公开类 → 工具库通常类较少，全量扫描
- 静态方法 vs 实例方法 → 使用方式
- 构造函数 / Builder → 初始化方式
- 方法参数和返回值 → API 设计风格

### 第三轮：特征探测
- 是否有状态（实例变量）→ 影响线程安全性
- 是否需要资源管理（`Closeable`）→ 使用注意事项
- 是否有性能敏感操作（大文件、批量处理）→ 性能说明
- 是否有 Spring 集成（`@Component` 等）→ 可选 vs 必须

## 问题清单

### AI 自答
1. 工具库解决什么问题？封装了什么能力？
2. 核心类有哪些？各自职责？
3. 每个类是否有状态？是否线程安全？
4. API 设计风格：静态方法 / Builder / 链式调用？
5. 依赖了哪些第三方库？版本？
6. 是否需要资源管理（需要 close）？
7. 有哪些典型使用场景？（从测试用例提取）
8. 是否有 Spring 集成？是可选还是必须？

### 用户决策点
- 文档侧重：快速上手 or 完整 API 参考？
- 是否需要每个方法都列出？
- 是否需要性能说明？

## 章节推荐池

### 核心章节
| 章节 | 内容 |
|------|------|
| 概述 | 功能简介、适用场景 |
| 快速开始 | Maven 依赖 + 最简使用示例 |
| 核心类 | 类表格（类名 / 职责 / 线程安全性） |
| API 参考 | 按功能分组的方法列表 |

### 条件章节
| 章节 | 触发条件 |
|------|---------|
| 资源管理 | 发现 `Closeable` 或资源释放逻辑 |
| 性能考量 | 发现大数据量处理或批量操作 |
| Spring 集成 | 发现 Spring 相关注解或配置 |
| 异常处理 | 发现自定义异常或明确的异常策略 |

### 可选章节
| 章节 | 适用场景 |
|------|---------|
| 使用示例集 | 场景丰富时 |
| 最佳实践 | 有常见陷阱时 |
| 与其他工具对比 | 有替代方案时 |
