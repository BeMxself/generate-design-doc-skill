# 前端组件库 — 分析指导

## 分析模板

### 第一轮：结构扫描
- `package.json` → 依赖、peerDependencies、构建脚本
- 目录结构 → 组件目录组织（按功能/按类型）
- 构建配置 → 打包方式（ESM/CJS/UMD）
- Storybook / 文档站配置 → 已有文档系统

### 第二轮：核心代码扫描
- 组件入口（`index.ts`）→ 导出清单
- 各组件目录 → 组件实现、Props 定义、样式
- 主题/Token 定义 → 设计变量
- 工具函数 → 内部辅助

### 第三轮：特征探测
- 设计 Token 系统 → 颜色/间距/字体变量
- 主题切换 → 暗色模式/自定义主题
- 无障碍支持 → ARIA 属性、键盘导航
- 国际化 → 组件内文案处理
- 动画系统 → 过渡/动画方案
- 表单集成 → 表单控件协议
- 按需加载 → Tree-shaking 支持

## 问题清单

### AI 自答
1. 组件库基于什么框架？（React/Vue/Web Components）
2. 包含多少个组件？按什么维度分类？
3. 组件的 Props/API 设计风格是什么？
4. 有设计 Token 系统吗？如何组织？
5. 支持主题切换吗？实现方式？
6. 样式方案是什么？（CSS-in-JS/CSS Modules/Less/Sass）
7. 有无障碍支持吗？覆盖程度？
8. 打包输出格式？支持按需加载吗？
9. 有 Storybook 或文档站吗？

### 用户决策点
- 文档侧重：组件使用指南 or 组件库架构？
- 是否需要列出每个组件的 Props？
- 是否需要包含设计规范说明？

## 章节推荐池

### 核心章节
| 章节 | 内容 |
|------|------|
| 概述 | 组件库定位、技术栈、组件数量 |
| 组件清单 | 按分类的组件表格 |
| 使用方式 | 安装、引入、基本用法 |

### 条件章节
| 章节 | 触发条件 |
|------|---------|
| 设计 Token | 发现 Token/变量定义 |
| 主题系统 | 发现主题切换机制 |
| 无障碍 | 发现 ARIA 支持 |
| 表单集成 | 发现表单控件协议 |
| 国际化 | 发现组件内 i18n |

### 可选章节
| 章节 | 适用场景 |
|------|---------|
| 组件开发规范 | 面向组件库贡献者时 |
| 构建与发布 | 构建流程有特殊处理时 |
| 迁移指南 | 有大版本升级时 |
