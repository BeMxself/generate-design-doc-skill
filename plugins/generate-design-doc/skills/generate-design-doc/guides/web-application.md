# 前端应用模块 — 分析指导

## 分析模板

### 第一轮：结构扫描
- `package.json` → 依赖、脚本命令、技术栈版本
- 目录结构 → 页面/组件/状态/路由/API 组织方式
- 构建配置 → webpack/vite/next.config 等
- `tsconfig.json` → TypeScript 配置和路径别名

### 第二轮：核心代码扫描
- 路由配置 → 页面结构和导航关系
- 页面组件 → 顶层页面及其职责
- 状态管理 → Store/Context/Hooks 定义
- API 层 → 接口调用封装方式
- 公共组件 → 复用的 UI 组件

### 第三轮：特征探测
- 状态管理方案 → Redux/Zustand/Pinia/Context
- 路由方案 → React Router/Vue Router/Next.js
- UI 框架 → Ant Design/Element/MUI/自研
- 国际化 → i18n 配置
- 权限控制 → 路由守卫、权限指令
- 主题/样式方案 → CSS Modules/Tailwind/Styled
- 数据请求 → Axios/Fetch/SWR/React Query
- 表单方案 → 表单库或自研方案

## 问题清单

### AI 自答
1. 技术栈是什么？（框架 + UI 库 + 状态管理 + 构建工具）
2. 页面结构如何组织？有哪些核心页面？
3. 路由结构是什么？有嵌套路由吗？
4. 状态管理方案是什么？Store 如何划分？
5. API 调用如何封装？有统一的请求/响应处理吗？
6. 有哪些公共组件？复用程度如何？
7. 权限控制如何实现？
8. 样式方案是什么？有主题系统吗？
9. 有国际化支持吗？
10. 构建和部署配置是什么？

### 用户决策点
- 文档侧重：架构概览 or 组件详细说明？
- 是否需要包含页面截图/交互说明？
- 是否需要包含组件 Props API？
- 哪些业务模块需要详细说明？

## 章节推荐池

### 核心章节
| 章节 | 内容 |
|------|------|
| 概述 | 技术栈、项目定位、核心功能 |
| 项目结构 | 目录组织和职责划分 |
| 页面与路由 | 页面清单、路由结构、导航关系 |

### 条件章节
| 章节 | 触发条件 |
|------|---------|
| 状态管理 | 发现 Store/Context 定义 |
| API 层设计 | 发现统一的请求封装 |
| 公共组件 | 发现 5+ 个复用组件 |
| 权限系统 | 发现路由守卫或权限控制 |
| 国际化 | 发现 i18n 配置 |
| 主题与样式 | 发现主题配置或设计 Token |

### 可选章节
| 章节 | 适用场景 |
|------|---------|
| 核心业务流程 | 复杂交互流程时 |
| 构建与部署 | 构建配置有特殊处理时 |
| 性能优化 | 有懒加载/代码分割等策略时 |
